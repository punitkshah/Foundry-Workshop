# Contoso Air: Microsoft Foundry Agent Lifecycle Workshop

This repository contains a complete customer-facing workshop for building AI agents with **Microsoft Foundry** and the **Microsoft Agent Framework** using **Python**.

## Workshop Goal

Participants build, evaluate, deploy, and consume AI agents for realistic airline operations scenarios:

1. **Build** a prompt-based agent in Foundry
2. **Evaluate** agent quality with practical datasets
3. **Deploy** the agent to a managed endpoint
4. **Consume** the deployed agent from Python
5. **Extend** to a multi-agent architecture with orchestration

## Audience

- Technical architects
- Developers
- AI engineers
- Solution architects
- Teams new to Microsoft Foundry and Microsoft Agent Framework

## Duration

- Introduction: 15 minutes
- Lab 1: 45-60 minutes
- Lab 2: 60-75 minutes
- Wrap-up and Q&A: 15 minutes

## Workshop Story

Contoso Air wants to improve operational resilience with AI agents.

- **Lab 1** solves a narrow but valuable problem: generating consistent flight delay communications.
- **Lab 2** evolves the solution into a coordinated disruption-management system with specialist agents.

5. Close with the comparison of prompt-based vs multi-agent patterns.

## Key Learning Outcomes

By the end of the workshop, participants can:

- Create agents in Microsoft Foundry
- Configure instructions and behavior
- Run and interpret evaluations
- Deploy agent versions
- Call deployed agents from Python
- Build agents with Microsoft Agent Framework
- Implement multi-agent orchestration
- Explain when to use prompt-based vs multi-agent designs
